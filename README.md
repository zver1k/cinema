# CinemaVision

Пет-проект на Next.js 16: каталог фильмов и сериалов с поиском, страницами тайтлов, авторизацией, профилем пользователя, избранным и списками просмотра.

Проект сделан как учебная, но приближенная к реальному продукту работа: здесь есть серверный рендеринг, работа с внешним API, база данных, авторизация, server actions, оптимистичный UI и разделение кода по зонам ответственности.

Автор: [Дамир](https://damirmedia.ru)

## Что умеет приложение

- Главная страница с подборками фильмов и сериалов со ссылками «Смотреть все».
- Каталог `/movies` с фильтрацией по жанрам, поиском и двумя режимами загрузки: пагинация через URL и бесконечная прокрутка.
- Страница новинок `/new` с бесконечной лентой (данные загружаются на сервере, показываются порциями через `IntersectionObserver`).
- Страницы `/top?type=movies` и `/top?type=tv-shows` — топ 250 фильмов и сериалов с бесконечной прокруткой через `useInfiniteQuery`.
- Поиск в шапке с debounce, выпадающими подсказками и переходом на страницу фильма.
- Детальная страница фильма:
  - постер, рейтинг, жанры, страны, режиссер, длительность и описание;
  - актеры, режиссеры и съемочная группа;
  - похожие фильмы;
  - рецензии, факты, бюджет и сборы;
  - блок "где смотреть";
  - кнопки "в избранное", "просмотрено" и "хочу посмотреть".
- Модальное окно фильма: клик на карточку из каталога открывает модалку с постером, рейтингом, жанрами и описанием поверх текущей страницы (parallel + intercepting routes). Прямой заход по URL открывает полную страницу.
- Отдельная страница рецензий `/movies/[id]/reviews` с пагинацией.
- Страницы персон `/person/[id]` с фильмографией.
- Регистрация, вход и восстановление пароля через Better Auth.
- Личный профиль:
  - счетчики избранного, просмотренного и запланированного;
  - вкладки "Избранное", "Просмотренные", "Хочу посмотреть", "Настройки";
  - сортировка фильмов по названию, году и рейтингу;
  - редактирование имени пользователя через форму на `useActionState` с серверной валидацией.

## Стек

- **Next.js 16** и **React 19**.
- **TypeScript**.
- **App Router**, React Server Components, streaming через `loading.tsx` и `<Suspense>`, границы ошибок `error.tsx`, `generateMetadata`, `connection`.
- **Server Actions** для мутаций, серверной валидации форм (`useActionState`) и серверных оберток над загрузчиками.
- **Route Handlers** для серверных эндпоинтов (Better Auth, on-demand сброс кэша).
- **Кэширование данных**: `revalidate` для загрузчиков внешнего API + точечная инвалидация по тегам (`revalidateTag`).
- **Prisma 7** и **PostgreSQL**.
- **Better Auth** для email/password авторизации; защита маршрутов через `proxy`.
- **TanStack Query v5** для клиентских сценариев: подсказки поиска и infinite scroll.
- **Tailwind CSS 4**, **shadcn/ui**, **Radix UI**, **lucide-react**.
- **Swiper** для подборок.
- **Vitest** для юнит-тестов.
- **Kinopoisk API Unofficial** как внешний источник данных.

## Архитектура

Код разложен в стиле, близком к Feature-Sliced Design:

```text
src/
├── app/        # маршруты Next.js, pages, layouts, server actions
├── widgets/    # крупные UI-блоки: Header, Sidebar, Search, подборки
├── shared/     # общие UI-компоненты, API-клиенты, типы, constants, utils
├── providers/  # провайдеры приложения
├── lib/        # серверные утилиты: auth, prisma, favorites, watchlist
└── generated/  # сгенерированный Prisma Client
```

В проекте разделены чтение и мутации:

- загрузчики данных лежат в `src/shared/api` и `src/lib`;
- мутации живут в `actions.ts` рядом с соответствующим маршрутом;
- серверные компоненты получают данные и передают в клиентские компоненты только сериализуемые props;
- интерактивные части вынесены в `"use client"` компоненты.

## Что демонстрирует проект

- Понимание Next.js App Router и новых особенностей Next.js 16: `params` / `searchParams` как `Promise`, серверные компоненты, streaming через `loading.tsx`.
- Умение выбирать место выполнения логики: сервер для данных и SEO, клиент для интерактива.
- Работа с внешним API и нормализация разных форматов данных под единый UI-контракт.
- Настройка авторизации, сессий и пользовательских данных через Better Auth + Prisma.
- Проектирование БД для пользовательских списков:
  - `Favorite` с уникальной парой `userId + movieId`;
  - `Watchlist` со статусами `PLANNED` и `WATCHED`;
  - каскадное удаление пользовательских данных.
- Оптимистичный UI для быстрых действий пользователя.
- URL как источник состояния для каталога: фильтры, поиск, пагинация, сортировка.
- Parallel + Intercepting Routes для модального окна без потери URL и SEO полной страницы.
- Использование TanStack Query там, где данные действительно живут на клиенте.
- Стриминг секций детальной страницы через `<Suspense>` и устранение водопадов данных (`Promise.all`).
- Границы ошибок `error.tsx` с разделением dev/prod-сообщений и `digest`.
- Стратегия кэширования внешнего API и экономия квоты: `revalidate`, `robots.ts`, on-demand инвалидация по тегам через Route Handler.
- Серверная валидация форм (`useActionState` + zod): ошибки возвращаются как значение и показываются пользователю.
- Переиспользуемые UI-компоненты: `MovieGrid`, `PosterImage`, `RatingBadge`, `GenreFilter`, `CollectionSwiper`.
- Обработка пустых состояний, 404 и неполных данных внешнего API.

## Переменные окружения

Создайте `.env` в корне проекта:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/cinema"

API_URL="https://kinopoiskapiunofficial.tech"
API_KEY="your_kinopoisk_api_key"

BETTER_AUTH_SECRET="your_secret"
BETTER_AUTH_URL="http://localhost:3000"

REVALIDATE_SECRET="your_revalidate_secret"
```

`REVALIDATE_SECRET` защищает эндпоинт `/api/revalidate`, который точечно сбрасывает кэш фильма по тегу.

Для деплоя также поддерживается `VERCEL_URL`: если `BETTER_AUTH_URL` не задан, приложение собирает base URL из переменной Vercel.

## Запуск локально

Проект использует `bun.lock`, поэтому основной вариант запуска через Bun:

```bash
bun install
bun --bun run prisma migrate dev
bun run dev
```

Приложение будет доступно на `http://localhost:3000`.

Альтернативно можно использовать npm:

```bash
npm install
npx prisma migrate dev
npm run dev
```

Схема находится в `prisma/schema.prisma`.

## Особенности реализации

### Server-first подход

Большинство страниц загружают данные на сервере. Это дает нормальный initial render, меньше клиентского JavaScript и более предсказуемую работу с секретами API.

### Клиентские сценарии там, где они нужны

TanStack Query используется для автодополнения поиска и бесконечной прокрутки. Секретный ключ Kinopoisk API при этом не попадает в браузер: клиент вызывает server action, а реальный запрос остается на сервере.

### Оптимистичные действия

Кнопки избранного и watchlist дают мгновенный отклик через `useOptimistic`, а серверная мутация синхронизирует состояние с PostgreSQL и инвалидирует страницу.

### Устойчивость к неполным данным API

Внешний API может возвращать разные поля для разных эндпоинтов и не всегда содержит бюджет, факты, актеров или похожие фильмы. Загрузчики нормализуют ответы и возвращают безопасные пустые состояния.

### Кэширование и экономия квоты

У внешнего API жесткий суточный лимит, поэтому загрузчики деталей кэшируются (`revalidate`), а краулеры отсекаются через `robots.ts`. Данные фильма помечены тегами, что позволяет точечно сбросить кэш конкретного тайтла через защищенный Route Handler `/api/revalidate`, не дожидаясь истечения таймера.

### Границы ошибок и стриминг

Непойманные ошибки сегментов перехватываются `error.tsx` (с разделением dev/prod-сообщений и `digest` для связи с логами). Тяжелые секции детальной страницы стримятся независимо через `<Suspense>`, а независимые запросы распараллелены `Promise.all`.

### Тестирование

Чистые утилиты покрыты юнит-тестами на Vitest (`bun run test`).

## Статус проекта

Проект активно развивается как pet-проект для закрепления Next.js, React, TypeScript и fullstack-подхода во фронтенд-разработке.

Планируемые улучшения:

- адаптивная доработка отдельных экранов;
- расширение покрытия тестами (компоненты и сценарии в дополнение к юнит-тестам утилит);
- полноценная отправка email для восстановления пароля;
- расширение настроек профиля;
- улучшение accessibility для интерактивных компонентов.
